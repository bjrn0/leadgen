## Project Overview

- **Type:** `web` (monolith as implemented; aspirational monorepo per README)
- **Primary Language:** TypeScript
- **Architecture:** Component-based Next.js App Router SPA shell with local state and mock data
- **Repository:** single implemented part (`app/`); `services/` and `database/` from the README do not exist yet

## ⚠️ Status in one line

The repo currently holds a **frontend prototype only** — a Next.js dashboard rendering the two-pillar product UI against hardcoded mock data. No backend, database, API, or AI/ingestion pipeline is implemented. The AWS-native engine in `specs/` is the **target**, not the current state. Details: [project-overview.md](./project-overview.md) and [architecture.md](./architecture.md#1-implemented-architecture-current-reality).

## Getting Started

1. `npm install` (root; installs the `app` workspace).
2. `cp .env.example .env.local` — for the UI, only `NEXT_PUBLIC_AUTH_MODE` matters (defaults to `dev-bypass`, no auth).
3. `npm run dev` and open http://localhost:3000.
4. Toggle **Lead Generation** / **Monitoring** in the sidebar. All actions are simulated via toasts.
