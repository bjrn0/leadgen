## Environment configuration

Copy the example file and fill in only what you need:

```bash
cp .env.example .env.local
```

For the **UI prototype**, the only variables that affect runtime are:

- `NEXT_PUBLIC_AUTH_MODE` â€” `dev-bypass` (default, no auth) or any other value to enable Clerk.
- `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` â€” required only when auth is enabled.

All other variables in `.env.example` (`DATABASE_URL`, `AWS_*`, `S3_BUCKET`, `PINECONE_*`, `COHERE_API_KEY`, `NEBIUS_*`, `FIRECRAWL_API_KEY`, `BRIGHTDATA_API_KEY`, `WEBSHARE_PROXY_URL`, `YOUTUBE_API_KEY`, `SLACK_WEBHOOK_URL`, `TELEGRAM_*`) belong to the **planned backend** and are not consumed by any current code.

## Enabling authentication (optional)

1. Set `NEXT_PUBLIC_AUTH_MODE` to something other than `dev-bypass`.
2. Provide `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` (and `CLERK_SECRET_KEY` for server features).
3. `layout.tsx` will then wrap the app in `<ClerkProvider>`.
